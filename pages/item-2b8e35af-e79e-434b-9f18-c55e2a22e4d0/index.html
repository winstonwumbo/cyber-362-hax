Navigating the Command Line

The command line interface (CLI), or terminal, is a powerful, text-based environment for interacting with your computer. CLI tools are extremely common in the IT industry across development, cybersecurity, and more. The CLI has remained a core component of every major operating system for decades. Becoming familiar with such tools will be an incredibly valuable, long-lasting skill. Although you may not be familiar with the term CLI, you've likely heard of some of the following software (which are all CLIs!):

  • Windows: Command Prompt, PowerShell
  • macOS: Terminal
  • Linux: Konsole, GNOME Console

The most common action that you'll perform in the command line will likely be 1. moving between folders and 2. listing folder contents.

1. Moving Between Folders

To move between folders, you will use the cd command. This same command is available on every operating system. cd is short for "Change Directory." Entering cd by itself (with no other text) will automatically bring you back to your home folder (/home/<username> on macOS/Linux, C:\Users\<username> on WIndows). You can use another command pwd to confirm where you're located. pwd is short for "Print Working Directory."

$ will represent separate lines of terminal commands.

When moving to a specific location, you can use a relative path or an absolute path. A relative path starts looking for the target folder from your current folder. This means that you don't need to fill in the file path (think of this as the folder sequence) from above your current folder. An absolute path is the full sequence of folders that takes you from the start of your hard drive (/ on macOS/Linux, C: on Windows) to the targeted folder.

Sometimes you may need to return back to a folder above your current location. To go backwards without typing an absolute path, the cd command accepts a sequence of characters (../) that symbolize "return up once."

2. Listing Folder Contents

To list folder contents, you can either use the ls command on macOS/Linux or the dir command on Windows. ls is short for "list," while dir is short for "directory." Certain software is available to emulate ls on Windows (Git Bash for example), if desired. These commands both provide a simple list of visible files and subfolders at your current location.

Sometimes you may need to view hidden folders or find detailed information like file size. This can be achieved with something called a CLI flag. Many CLI tools allow you to pass in special options to change behaviors. 

For the ls command, the correct sequence would be ls -lsa. For the dir command, the correct sequence would be dir /a /q. Each of these makes a request for a more advanced output.
